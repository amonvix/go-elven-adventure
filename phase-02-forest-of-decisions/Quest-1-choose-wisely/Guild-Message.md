# ⚔️ Phase 02 — Forest of Decisions

## Quest 1: The First Choice

The path through the Forest of Decisions is not linear.

For the first time, your journey will no longer be guided only by what *is*…  
but by what *could be*.

The Guild Master raises his staff:

> "A hero is not defined by what he holds,  
> but by the choices he makes when the path divides."

---

## Objective

Understand how to compare values and make decisions using conditional logic.

---

## Concepts Learned

- comparison operators
- boolean expressions
- if statement

---

## Your Task

Create a new file:

decision.go

Inside it, declare the following variables:

- heroName (string)
- heroLevel (int)
- heroAlive (bool)

Then create a decision using `if`:

If the hero is alive, print a message saying the journey continues.  
If not, print that the journey has ended.

---

## Example Inspiration

If heroAlive is true:

The journey of Amon continues through the forest.

If heroAlive is false:

The journey ends here. The forest claims another soul.

---

## Side Quest

Add a second condition based on level.

If the heroLevel is greater than 10:

The forest recognizes a seasoned warrior.

If not:

The forest sees an unproven traveler.

---

## Mini Boss Fight

The forest tests your judgment.

Create a situation where BOTH conditions matter:

- hero must be alive
- hero must have level greater than 5

If both are true:

Allow the hero to pass deeper into the forest.

If not:

Block the path.

---

## Expected Thinking

You will combine conditions to control the outcome of your program.

---

## Reward Unlocked

- You can compare values
- You can control program flow
- You begin to simulate real decision-making

---

## Victory Condition

- Program runs successfully
- At least one if condition implemented
- Side Quest completed
- Mini Boss condition works correctly
